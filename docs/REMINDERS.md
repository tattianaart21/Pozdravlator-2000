# Напоминания в Telegram и VK

Чтобы пользователь получал напоминания о днях рождения и событиях там, где он всегда онлайн (Telegram, VK), нужна интеграция с мессенджерами.

## Варианты реализации

### 1. Telegram Bot

- Создать бота через [@BotFather](https://t.me/BotFather).
- Пользователь привязывает аккаунт: отправляет боту команду или ссылку из веб-приложения с токеном/кодом.
- Бэкенд (Supabase Edge Function или отдельный сервис) по расписанию проверяет предстоящие события и отправляет напоминания через [Telegram Bot API](https://core.telegram.org/bots/api) (метод `sendMessage`).
- Хранить в БД: `user_id` (Supabase Auth) и `telegram_chat_id` (получают после того, как пользователь написал боту).

### 2. VK

- [VK API](https://dev.vk.com/) позволяет отправлять сообщения от имени группы при наличии разрешения пользователя.
- Пользователь подключает уведомления через OAuth VK; сохраняем `user_id` VK.
- Cron или Edge Function раз в день проверяет события и отправляет напоминания через [messages.send](https://dev.vk.com/method/messages.send).

### 3. Единый бэкенд

- В Supabase: таблица `user_reminders` (user_id, channel: 'telegram'|'vk', external_id, enabled).
- Edge Function с триггером по расписанию (Supabase pg_cron или внешний cron): для каждого пользователя с включёнными напоминаниями за N дней до события вызывать Telegram/VK API.

## Что уже есть в приложении

- В календаре есть кнопки «Добавить в Google Calendar» и «Скачать .ics» — пользователь может подписаться на события в своём календаре и настроить напоминания там.
- Ссылки «В Telegram» и «В VK» при копировании поздравления открывают шаринг готового текста; не отправляют напоминания автоматически.

Полноценная отправка напоминаний в мессенджеры требует бэкенда с хранилищем привязок (user ↔ telegram_chat_id / vk_id) и периодического job’а для рассылки.
